## ما هو المرتجع؟

**المرتجع** هو عندما يُعيد عميل منتجات من طلب مكتمل. تتتبع المرتجعات العناصر المُرتجعة وسببها وكيفية معالجة الاسترداد.

## أنواع المرتجعات

كاشفيو يدعم ثلاثة أنواع من المرتجعات:

| النوع | الوصف |
|-------|-------|
| **مرتجع واسترداد** | إرجاع فعلي للمنتجات + استرداد للعميل |
| **مرتجع فقط** | إرجاع فعلي للمنتجات، لا يتم إصدار استرداد |
| **استرداد فقط** | استرداد للعميل دون الحاجة لإرجاع فعلي |

عند إنشاء مرتجع، تختار النوع المناسب بناءً على احتياجات عملك ووضع العميل.

## إنشاء مرتجع

تُنشأ المرتجعات من طلب محدد:

1. افتح صفحة **تفاصيل الطلب** للطلب الذي تريد إرجاع عناصر منه.
2. انقر على **إنشاء مرتجع**.
3. اختر العناصر للإرجاع:
   - حدد العناصر التي يُرجعها العميل
   - أدخل **الكمية** المُرتجعة لكل عنصر
   - اختر **سبب الإرجاع** لكل عنصر

### أسباب الإرجاع

عند إنشاء مرتجع، يجب اختيار سبب لكل عنصر. الأسباب المحددة مسبقاً المتاحة هي:

| السبب | الوصف |
|-------|-------|
| **تغيير الرأي** | قرر العميل عدم الاحتفاظ بالمنتج |
| **منتج خاطئ** | تم توصيل منتج مختلف |
| **عيب** | المنتج به عيب تصنيع |
| **تالف** | المنتج تعرض للتلف أثناء التوصيل أو المناولة |
| **لا يطابق الوصف** | المنتج لا يتطابق مع وصفه |
| **مشكلة في المقاس** | مقاس خاطئ أو لا يناسب |
| **مشكلة في الجودة** | جودة المنتج أقل من المتوقع |
| **طلب مكرر** | طلب العميل بالخطأ مرتين |
| **أجزاء ناقصة** | المنتج غير مكتمل أو أجزاء مفقودة |
| **منتهي الصلاحية** | المنتج تجاوز تاريخ صلاحيته |
| **سبب آخر** | أي سبب آخر (أضف ملاحظة) |

### أسباب عدم إعادة التخزين

عند معالجة مرتجع، يمكنك وضع علامة على العناصر كـ **غير قابلة لإعادة التخزين** إذا لم يمكن إرجاعها للمخزون. أسباب عدم إعادة التخزين المتاحة هي:

| السبب | الوصف |
|-------|-------|
| **تالف** | العنصر تالف جداً ولا يمكن إعادة بيعه |
| **عيب** | العنصر به عيوب تمنع إعادة البيع |
| **منتهي الصلاحية** | العنصر تجاوز تاريخ صلاحيته |
| **ملوث** | العنصر ملوث ولا يمكن إعادة بيعه |
| **تعويض** | العنصر أُعطي كتعويض للعميل |
| **العميل احتفظ به** | العميل احتفظ بالعنصر رغم طلب الإرجاع |
| **مشكلة في الجودة** | مشاكل الجودة تمنع إعادة البيع |
| **التغليف تالف** | التغليف تالف والعنصر لا يمكن إعادة بيعه |
| **مخزون غير قابل للتتبع** | العنصر غير متتبع في نظام المخزون |
| **سبب آخر** | أي سبب آخر (أضف ملاحظة) |

4. راجع ملخص المرتجع.
5. انقر على **إرسال** لإنشاء المرتجع.

## إدارة المرتجعات

اذهب إلى **المبيعات > المرتجعات** في القائمة الجانبية لرؤية جميع المرتجعات.

قائمة المرتجعات تعرض:

| العمود | الوصف |
|--------|-------|
| **رقم المرتجع** | معرف فريد للمرتجع |
| **رقم الطلب** | الطلب الأصلي |
| **العميل** | من يُرجع العناصر |
| **الحالة** | قيد الانتظار، معتمد، مرفوض، أو ملغي |
| **التاريخ** | متى تم إنشاء المرتجع |
| **المبلغ** | قيمة المرتجع |

## حالات المرتجع

| الحالة | المعنى |
|--------|--------|
| **قيد الانتظار** | تم إنشاء المرتجع، في انتظار المراجعة |
| **معتمد** | تم قبول المرتجع، سيتم إعادة تخزين العناصر |
| **مرفوض** | تم رفض المرتجع |
| **ملغي** | تم إلغاء المرتجع |

## اعتماد أو رفض مرتجع

1. افتح تفاصيل المرتجع.
2. راجع العناصر والأسباب.
3. انقر على **اعتماد** أو **رفض**.

عند **اعتماد** المرتجع:
- يتم استعادة مخزون العناصر المُرتجعة (إذا كان تتبع المخزون مفعّلاً والعناصر معلمة كقابلة لإعادة التخزين)
- العناصر المعلمة بأسباب عدم إعادة التخزين **لا** تُضاف للمخزون
- يمكن إصدار استرداد من تفاصيل المرتجع أو الطلب (إذا كان نوع المرتجع يتضمن استرداد)

<Callout title="ملاحظة">
المرتجعات والاستردادات إجراءات منفصلة. اعتماد المرتجع لا يصدر استرداداً تلقائياً. تحتاج لإنشاء استرداد بشكل منفصل إذا أردت إرجاع المال للعميل.
</Callout>
