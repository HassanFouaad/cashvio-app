title: Users & Invitations
description: How to invite team members, assign roles, and manage user accounts in your Cashvio workspace.
---

## Managing Users

Go to **Users** in the sidebar to see all team members in your Cashvio workspace.

The users list shows:

| Column | Description |
|--------|-------------|
| **Name** | User's full name |
| **Email** | User's email address |
| **Role** | The role(s) assigned to this user |
| **Status** | Active or Inactive |
| **Last Login** | When the user last logged in |

## Inviting a New User

1. Go to **Users** in the sidebar.
2. Click **Invite User**.
3. Fill in the invitation details:

| Field | Description |
|-------|-------------|
| **Email** | The email address of the person you want to invite |
| **First Name** | Their first name |
| **Last Name** | Their last name |
| **Role** | The role to assign (determines their permissions) |
| **Store** | Which store(s) they can access (if applicable) |

4. Click **Send Invitation**.

The invited person receives an email with a link to set up their account and join your workspace.

## User Details

Click on a user to see their full profile:

- **Personal Information** — Name, email, and contact details
- **Role** — Current role assignment
- **Store Access** — Which stores they can access
- **Permissions Summary** — Overview of what they can do
- **Activity** — Recent login and activity information

## Editing a User

1. Open the user's details page.
2. Click **Edit**.
3. You can update:
   - Personal information
   - Role assignment
   - Store access
4. Click **Save Changes**.

## Activating and Deactivating Users

- **Deactivate** a user to prevent them from logging in without deleting their account.
- **Activate** a user to restore their access.

This is useful when team members temporarily leave or change positions.

## Deleting a User

1. Open the user's details page.
2. Click **Delete** from the actions menu.
3. Confirm the deletion.

<Callout title="Warning">
Deleting a user removes their access permanently. Their historical actions (like orders they created) are preserved in the system logs.
</Callout>
